# 多线程

为什么需要多线程？这个问题问得很基础，但也很核心。简单来说，多线程是为了榨干 CPU 的性能，同时让程序响应得更快。当然，多线程也带来了新麻烦比如资源竞争、死锁，但为了效率和体验，这些代价通常是值得的。

# 守护线程

守护线程是后台保姆线程，当所有非守护线程执行完毕后，JVM 会自动退出并强制终止所有守护线程。

线程 1 执行完 50 次循环后，JVM 会立即退出，此时线程 2 是守护线程即使才执行到一半，也会被强制终止，控制台不会打印完所有线程 2 的输出。

synchronized 锁的是对象本身：非静态方法锁 this ，有多少个对象就有多少把锁；静态方法锁 类.class ，所有对象只有一把锁。代码只创建了一个 TicketTast 实例，所以只有一把锁，两个窗口争抢这一把锁，安全有效。

# 线程池

线程池的核心作用是：复用线程、控制并发、提升性能。

1. 复用线程（降低开销）
创建和销毁线程是有成本的（内存分配、系统调用）。线程池让线程执行完一个任务后不销毁，而是继续执行下一个任务，避免了频繁创建销毁的开销。

2. 控制并发（防止资源耗尽）
没有线程池时，你可以无限制地创建线程（比如循环创建 10000 个），系统可能崩溃。线程池限制了最大线程数，超过的任务进入队列排队，保护系统稳定。

3. 统一管理（方便监控）
线程池提供了任务统计（执行数、失败数）、定时执行、线程状态监控等功能，方便统一管理。

没有线程池：每次来客人都临时招聘一个服务员，客人走了就辞退 —— 频繁招聘培训，成本极高。

有线程池：店里固定养了 10 个服务员，客人来了直接服务，走了继续服务下一位 —— 高效、可控。

# 注解

注解本质上是一种元数据（即 "数据的标签"），它本身不包含业务逻辑，只是给代码打上标记。真正起作用的是解析注解的处理器。

# 元注解

# @Retention（保留策略）

指定注解在什么阶段保留。

取值 保留阶段 说明
SOURCE 源代码 编译时丢弃，只存在于 .java 文件中
CLASS 字节码 编译到 .class 文件，但运行时不可用（默认值）
RUNTIME 运行时 编译到 .class ，运行时可通过反射获取

# @Target（适用目标）

指定注解可以用在什么地方。

取值 目标
TYPE 类、接口、枚举
FIELD 字段（包括枚举常量）
METHOD 方法
PARAMETER 方法参数
CONSTRUCTOR 构造函数
LOCAL_VARIABLE 局部变量
ANNOTATION_TYPE 注解类型
PACKAGE 包
TYPE_PARAMETER 泛型类型参数（Java 8）
TYPE_USE 类型使用处（Java 8）
