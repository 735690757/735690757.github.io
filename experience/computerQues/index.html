# 状态栏日历 / 通知中心无法打开

经过我的了解，一般来说这个问题是误使用 Dism++ 的配置所导致的，然而我并不是因为这个，因此这个方法对我来说没什么用。经过查阅官方的 bug 反馈渠道，许多人也同样遇到了这个问题，但是都没有得到解决，官方的重注册方法在我这里也是无效的。

这个问题多表现于更新之后发生的，通常问题的形式如下

  1. 更新 windows11 后，点击右下角时间，没有日历弹出
  2. 右下角点击无反应，不弹出调节框
  3. 桌面右下角的时间点击后无法弹出日历
  4. win11 专业版，任务栏右下角日历和通知按钮左键点击无反应，右键正常

而官方的回答多为：

  1. DISM++ 问题

  2. 创建一个新的本地账户再次确认问题

  3. 使用命令检查应用包集是否已损坏

  4. 重注册

  5. 重装系统

当然以上方法对于大多数人可能是可行的，但是在我这里完全没用。

# 解决方法

  1. 打开注册表编辑器

  2. 分别索引至

    和

  3. 他们下面有：DisableNotificationCenter，类型为 REG_DWORD

  4. 将其设置为 0

  5. 重启

这个困扰我两个月的问题解决了

对于官方那个新建用户的验证方法我觉得是有必要的，我尝试了，同样在新用户中也无法打开。注意到注册表中有两个重要分区，一个是 \HKEY_CURRENT_USER 另一个是 \HKEY_LOCAL_MACHINE ，正如其名一个是本机全局配置，另一个是 hi 当前用户的配置，我在修改的时候注意到，HKEY_CURRENT_USER 并没有设置为 1，只有 HKEY_LOCAL_MACHINE 被设置为 1 了，这也就证明了为什么新建用户依然无法打开日历了，因为 HKEY_LOCAL_MACHINE 起到了主导作用。

# 安全中心被禁用

同样这个问题也是在更新之后发生的，也是两个月左右。

依然导航到注册表 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\wscsvc 这个位置，将这个目录下的 Start 值修改为 2，2 代表自动，我之前是 4，4 的意思的禁用。

这里可能会遇到一个问题就是无权写值，这里通过巧妙的夺权操作来实现。

  1. 邮件左侧栏目的 wscsvc
  2. 点击权限
  3. 点击高级
  4. 点击所有者右侧的更改
  5. 在下面输入你这个用户的用户名
  6. 点击确认
  7. 再次确认
  8. 在组或用户名中选择当前登录的有权用户 / 组，一般为 Administrators（电脑名 \Administrators）
  9. 在下面的权限中的完全控制中选择允许
  10. 确定（至此夺权成功）
  11. Start 值修改为 2
  12. 重启

然后你的安全中心又可以正常使用了，虽然我经常会用一个脚本将他禁用，但是那也必须是我自己禁用的，我想开的时候就开，想关就关哈哈哈哈。至此啊，也算是解决了，我桌面上的 Chiikaawa 日历直接原地失业哈哈哈哈。
